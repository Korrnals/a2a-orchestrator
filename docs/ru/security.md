# Безопасность

Модель безопасности, threat landscape и руководство по усилению для
`a2a-orchestrator`.

## Модель безопасности

Оркестратор выполняет шесть проверок маршрутизации (R1–R6) при каждом
вызове `send_a2a`. Вместе они предотвращают наиболее частые failure-режимы
мультиагентных систем: циклы, неограниченную делегацию, несанкционированную
маршрутизацию и неподписанные сообщения от недоверенных агентов.

| Контроль | Правило | Что предотвращает |
| --- | --- | --- |
| Белый список | R1 | Несанкционированная маршрутизация между агентами |
| Защита от циклов | R2 | Зацикливание агентов друг на друге |
| Лимит глубины | R3 | Неограниченные цепочки делегирования |
| Лимит бюджета | R4 | Разрастание числа вызовов |
| Подпись | R6 | Подмена личности в распределённых развёртываниях |
| Согласие на деструктивное | R5 | Необратимые действия без одобрения пользователя |

## Threat model

| Угроза | Митигация |
| --- | --- |
| Подмена агента | R6 — проверка подписи Ed25519 |
| Цикл маршрутизации (A→Б→A) | R2 — обнаружение upstream в цепочке |
| Неограниченная рекурсия | R3 + R4 — лимиты глубины и бюджета |
| Несанкционированный доступ | R1 — белый список `accepts_routes_from` |
| Деструктивное действие без согласия | R5 — fail-closed провайдер согласия |
| Утечка данных между тенантами | `TenantManager` — полная изоляция по тенантам |
| Недоступность Mnemos | JSONL-fallback — долговременный локальный аудит-след |
| Повтор старых сообщений | Уникальный `message_id` на вызов; TTL nonce при регистрации |

## Усиление

### Продакшен-развёртывание

- **Задайте `A2A_WEB_API_KEY`** — REST-обёртка по умолчанию без auth.
  Всегда устанавливайте API-ключ при экспозиции web-сервера.
- **Ограничьте CORS** — установите `A2A_WEB_CORS_ORIGINS` на конкретные
  origin'ы, а не на `localhost` по умолчанию.
- **Используйте TLS** — WS и web-сервер по умолчанию используют plain
  HTTP/WS. Поставьте их за reverse-proxy (Caddy, nginx) с TLS-терминацией.
- **Зафиксируйте `A2A_CARDS_DIR`** — не полагайтесь на автоопределение в
  продакшене. Задайте переменную явно.

### Управление ключами

- Генерируйте ключевые пары Ed25519 с безопасным RNG
  (`Ed25519PrivateKey.generate()`).
- Храните закрытые ключи в secret-менеджере, никогда в Agent Card или
  закоммиченных файлах.
- Периодически ротируйте ключи; обновляйте поле `public_key` в Agent Card
  или перерегистрируйте агента.
- `KeyStore` хранит только **открытые** ключи — закрытые никогда не попадают
  в оркестратор.

### Провайдер согласия (R5)

Провайдер согласия по умолчанию **fail-closed** — он отклоняет все
деструктивные действия, если явно не переопределён. Интеграция VS Code UI
может подменить `consent_provider` в рантайме для интерактивных диалогов.
Никогда не заменяйте его на fail-open провайдер в продакшене.

## Чувствительные данные

Оркестратор не логирует payload'ы сообщений на уровне `INFO`. На уровне
`DEBUG` полные сообщения могут появиться в логах — используйте этот
уровень только в разработке. См. политику
[sensitive-data](https://github.com/Korrnals/GithubCopilotWorkflow)
проекта для полных правил редукции.
